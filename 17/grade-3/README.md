# Оценка 3 — Простые unit-тесты на `assert()`

## Идея

Без фреймворка — обычный `assert.h`. Цель — показать сам принцип unit-тестирования: подать на вход данные, сравнить результат с ожиданием, ткнуть `assert()` если не сошлось.

## Что тестируется

Функция `te_interp()` из библиотеки **tinyexpr** — берёт строку с математическим выражением и возвращает `double`.

| Категория | Примеры |
|---|---|
| Арифметика | `"1+2"`, `"100/4"` |
| Приоритет | `"2 + 3 * 4"` vs `"(2+3)*4"` |
| Степень | `"2^10"` = 1024 |
| Функции | `"sqrt(16)"`, `"sin(0)"` |
| Ошибки | `"1 +"`, `"foo"`, `"(1 + 2"` — должны выдать `err != 0` |

## Запуск

```bash
make run
```

Ожидаемый вывод (фрагмент):

```
OK:   "1+2" = 3
OK:   "10 - 4" = 6
...
OK:   "(1 + 2" — error at position 6 (as expected)

All 12 tests passed.
```

> Программа возвращает exit-код 0 при успехе. Любой `assert()` → `abort()` + ненулевой код.

## Почему `assert()` подходит только для оценки 3

`assert()` останавливает программу на первой ошибке. На больших наборах теряется картина «сколько прошло / сколько упало». Для серьёзных тестов лучше фреймворк — см. [grade-4](../grade-4) (CMocka).
